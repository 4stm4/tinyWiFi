# Changelog

## v0.1.0 — 2026-06-01

Первый релиз TinyWifi MVP.

### Ядро (`tinywifi-core`)
- Проверки доступности файлов, сервисов и сетевых интерфейсов перед любыми
  действиями (правило «проверь до того как трогать»).
- Парсер `hostapd.conf` (line-preserving, round-trip комментариев и
  неизвестных директив) и редактирование Wi-Fi с валидацией.
- Парсер `nanodhcp.conf` в реальном формате `key=value` с сохранением
  неизвестных ключей при записи.
- Чтение лизов, метрики хоста (`/proc`), модель системного статуса.
- Безопасные правки: backup → запись → перезапуск → проверка → откат;
  верификация запуска с ретраями; примитив `AutoRevert` (commit-confirm).
- Кросс-платформенный сервис-слой: systemd / SysV-init (`/etc/init.d`) /
  процессы.

### Веб-панель (`tinywifi-web`)
- Дашборд с живыми данными, страница System (статусы, рестарт, reboot).
- REST API: status, wifi, dhcp, leases, services, reboot.
- Commit-confirm с автооткатом (`?hold=` + `/confirm`).
- 10 интеграционных тестов HTTP-слоя.

### Дисплей (`tinywifi-display`)
- Демон статуса с абстракцией рендера (`Renderer`), мягкая деградация полей.

### Развёртывание
- Проверено на стенде «4stm4 netOS» (Buildroot, aarch64, busybox-init):
  реальные hostapd/nanodhcp, веб-панель на порту 80.

### Известные ограничения
- Кнопки Restart работают только для сервисов с отдельным init-скриптом; на
  netOS весь AP-стек ведёт мастер-скрипт `S10tinywifi`.
- Схема `[ap]/[network]/[dhcp]` из заготовки netOS-конфига пока не
  поддерживается (используется собственная `[web]/[paths]/[services]`).
